# Appark「图书查询器」市场调研

调研日期：2026-07-12  
目标市场：美国 iOS  
判断门槛：月下载量 >= 5,000，月收入 >= $200，优先观察近两年上架产品。

## 结论

最适合当前网站继续切入的方向不是泛化的「阅读记录」，而是：

> **输入书名、作者或 ISBN，立即判断是否存在合法免费电子版，并提供可用格式的直接下载或在线阅读入口。**

理由：

1. 阅读记录赛道已经被 Bookly、Bookmory、Margins 等高评分产品验证，但功能和竞争都很成熟，继续做需要账户、同步、统计、提醒和社区，偏离轻量单功能路线。
2. 公版书搜索下载已有 `eBook Search`（9,891 下载 / $1,974 收入）和 `My Books`（8,010 / $1,000）达到门槛，证明需求与付费存在。
3. 该赛道代表产品普遍较旧：`eBook Search` 2011 年上架、2024 年后未更新；`My Books` 2010 年上架、2020 年后未更新。搜索质量、格式提示、版权状态和界面都有优化空间。
4. 现有网站已经具备 Google Books 查询与 Gutenberg 聚合基础，增量开发最小，适合快速发布验证。

## 数据说明

- Appark 高级搜索已登录，但账户接口返回 `Current plan does not support this feature`，因此没有把高级搜索默认榜单误当成筛选结果。
- 数据改为通过 Appark 美国区单应用详情页逐个核验，并用美国 App Store Search API 交叉确认应用、评分和上架时间。
- 下表下载量和收入额为 2026-07-12 Appark 详情页默认统计周期的估算值。Appark 属于第三方估算，不等同于开发者后台实收。
- `达标` 表示同时满足下载量 >= 5,000、收入 >= $200。

## 可利用的真实差评

BookBuddy 的美国区近期低分评论集中在：

- 扫描后仍有不少书需要手工录入。
- 免费版只允许添加 50 本，用户录入后才遇到付费墙。
- 老书 UPC 和旧版 ISBN 识别不佳。
- 标签搜索与自定义标签层级不足。
- 频繁弹出评价提示，干扰连续录入。

这些抱怨说明「扫码识别 + 本地无限建库 + 清晰收费」是可验证的备选切口，但不是本轮首选。

## 推荐 MVP

产品一句话：**查一本书，告诉用户哪里能合法免费读，并直接给出可下载格式。**

首版只做以下闭环：

1. 支持书名、作者、ISBN 搜索；移动端支持扫码输入 ISBN。
2. Google Books 返回书目信息、封面、作者、出版时间和简介。
3. Gutendex / Project Gutenberg 等来源返回明确可用的 EPUB、HTML、TXT 等格式。
4. 每条结果显示 `可下载`、`仅在线阅读`、`暂无免费版本`，避免把预览链接伪装成下载。
5. 点击格式即可下载或打开原始来源，不要求登录。
6. 搜索历史和收藏仅保存在浏览器本地，首版不做账号和云同步。
7. 无合法免费版本时提供购买/借阅出口，后续可接图书联盟链接。

首版不做：阅读器、阅读时长统计、社区、书评、跨设备同步、复杂推荐、来源不明的版权书下载。

## 差异化与变现

差异化：

- 不是另一个书目数据库，而是回答「这本书现在能不能合法免费下载」。
- 对每个格式和来源说明版权/公版状态、文件类型与语言。
- 中文搜索结果与英文公版资源统一展示，减少用户在多个站点之间查找。
- 当没有免费版本时明确说没有，不制造无效下载按钮。

变现顺序：

1. 无免费版本时接 Amazon、Bookshop 或二手书平台联盟链接。
2. 页面底部轻广告，不遮挡搜索和下载。
3. 后续增加一次性 Pro：批量 ISBN 查询、CSV 导出、无广告；不急于做订阅。

## 7-14 天验证方案

1. 第 1-3 天：补齐合法下载来源聚合、结果状态和格式下载按钮。
2. 第 4-5 天：加入移动端 ISBN 扫码与本地搜索历史。
3. 第 6-7 天：上线 20-50 个长尾落地页，例如 `download public domain epub`、`ISBN free ebook lookup`。
4. 第 8-10 天：邀请朋友完成真实查书任务，记录无结果查询和错误来源。
5. 第 11-14 天：根据搜索日志补同义词、语言过滤和失败提示，只修影响闭环的问题。

验证门槛：

- 至少 100 个独立访客或 50 位真实测试用户。
- 搜索后出现有效书目结果的比例 >= 70%。
- 有合法资源的查询中，下载/阅读点击率 >= 20%。
- 无结果后继续修改关键词再次搜索的比例 >= 15%，说明任务真实而非随手访问。
- 至少 10 条明确反馈指向缺失来源、格式或语言，作为下一轮迭代依据。

## 最终排序

1. **首选：合法免费电子书查询与直接下载。** 现有工程复用最高，老竞品明显，网页分发和 SEO 更适合。
2. **备选：ISBN 扫码 + 本地无限藏书库 + CSV 导出。** 需求和付费强，但要解决识别率与批量录入体验。
3. **观察：二手书价格/价值查询。** 新产品已有收入信号，但尚未达到下载门槛，并且数据源稳定性决定成败。

不建议把 `reading tracker` 作为当前第二个单功能网站：市场很大，但已经不是一个轻量查询工具，而是完整 App 产品竞争。
