# Shelfmark Design Read

## Final visual target (locked)

本轮视觉目标已锁定为“Reading Portal + Infinite Index result panel”：

- 首屏左侧是由真实书页与硬壳构成的沉浸式阅读门户，作为 Shelfmark 的品牌记忆点。
- 右侧是结构化的选中书籍结果区，优先展示真实封面、作者、来源和合法阅读路径。
- 色彩改为黑樱桃、深酒红、墨黑、冰蓝文字与电珊瑚行动色；不再使用暖纸张、深森林绿或 P1 星空语言。
- 仍只保留一个首屏主 CTA：`Find this book`。预览、借阅和购买是结果路径，不伪装成产品营销功能卡。
- 门户背景、书封和品牌标记均为独立真实图像资产；React 负责可访问文本和交互，动效只承担空间关系与状态反馈。

## 产品判断

- 页面类型：Hybrid，首屏负责解释与搜索，结果区和私人书架属于高密度 App UI。
- 核心场景：用户带着一本具体的书来到网站，输入一次，在一分钟内选择合法阅读路径。
- 视觉锚点：真实书封、访问状态和下一步操作组成的结果预览；不是装饰插画。
- 可信度：来源标签、版权/可用性措辞、落地页域名和不夸大的限制说明。

## 产品身份

Shelfmark 的品牌来自“图书馆索书标记”和“阅读桌面”：温暖纸张、深墨绿色、克制的朱红索引线、紧凑的目录排版。页面应像一本高质量参考目录，而不是绿色 SaaS 模板。

P1“此刻星空”只提供完成度基线：强首屏、真实结果锚点、完整状态、三视口验收和克制动效。Shelfmark 不继承星空、液态玻璃、灰阶电影感或 P1 布局。

## 首屏构图

- 顶部：简洁品牌、`Find` 与 `My shelf` 两个视图入口、数据透明说明。
- 主区左侧：短标题、单一搜索输入、搜索模式与示例。
- 主区右侧：阅读路径说明和本地书架摘要，视觉重量低于搜索结果但高于营销文案。
- 唯一主 CTA：`Find this book`。

## 结果体验

- 搜索结果以“封面 + 书目 + access badge + 主行动”组织，禁止让用户先打开详情才知道是否能读。
- 公版下载、借阅、预览、购买和仅元数据必须用文字与图标共同区分。
- 详情抽屉/面板提供版本、来源、简介、保存到书架和复制信息。
- 任何免费状态都不得根据出版年份猜测，必须来自上游明确字段。

## 私人书架

- 搜索结果可保存为本地书架条目。
- 用户自有 EPUB/PDF 通过 IndexedDB 保存在当前设备；展示文件类型、大小和导入日期。
- 页面明确说明“不上传、不创建账户、清除浏览器数据会移除文件”。

## 响应式

- 1440×900：宽屏目录工作台；首屏双栏，结果列表保留直接行动，书架与详情形成稳定分区。
- 768×1024：单列但保留顶部视图切换；搜索和结果优先，透明说明后置。
- 390×844：标题压缩，搜索按钮全宽；结果卡主行动始终可见，次要元数据折叠；书架导入区不横向溢出。

## 状态

必须实现 default、hover、focus-visible、active、disabled、loading、empty、success、error、offline/fallback、local-storage-unavailable。

## 反模板约束

- 不使用通用三功能卡首屏、紫蓝渐变、全页居中或重复大圆角。
- 不把每个文本段落装进卡片；边界只服务真实交互与信息分组。
- 不用 CSS 画书、emoji、手绘 SVG 或无来源装饰图。
- 不复制 P1 的星空、严格灰阶、Poppins/Source Serif 4、液态玻璃或电影字幕。

## 截图断言

1. 1440×900 首屏清楚出现产品、输入、结果承诺和唯一主 CTA。
2. 390×844 没有横向滚动，搜索与书架导入触控目标不小于 44px。
3. 搜索完成态不打开详情也能识别合法访问方式。
4. 详情态可看到来源、限制、保存与阅读行动。
5. 本地书架同时展示保存记录和用户文件，并说明隐私边界。
