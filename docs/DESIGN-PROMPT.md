# Shelfmark 产品专属设计 Prompt

## Design Read

- 产品类型：Hybrid 工具网站。
- 目标用户：希望快速找到合法电子版、借阅、预览或购买入口的英语读者。
- 核心场景：用户输入书名、作者或 ISBN，比较不同来源提供的阅读路径，再把结果或自有文件放进本地书架。
- 主要输入：1–160 字符书名、作者或 ISBN。
- 主要结果：带真实封面、访问状态、来源与直接行动的书籍结果。
- 商业目标：验证购买联盟点击与专业书架/批量导出的付费意愿。
- 可信度来源：Open Library、Google Books、Gutendex 的明确字段与可核验落地页。
- 强视觉锚点：真实书封和阅读路径结果预览。
- 品牌记忆：图书馆 shelfmark、温暖纸张、深墨绿、细朱红索引线和编辑式目录节奏。
- 应避免：绿色 SaaS 模板、三卡片 Hero、空洞功能营销、把受限预览包装成免费下载。

## 设计目标

为 Shelfmark 设计并实现一个高可信、强记忆、可在一分钟内完成主任务的英语工具网站。用户在 5 秒内应理解：输入一本书，得到合法阅读路径，并可存到当前设备的私人书架。首屏只有一个主操作 `Find this book`。

## 信息架构

1. 首屏：品牌导航、明确标题、主要搜索、阅读路径图例和本地书架摘要。
2. 输入：书名/作者与 ISBN 两种模式共用一个真实标签输入框，带示例和本地条码扫描降级。
3. 处理反馈：在原位置显示检索来源和进度，不用全屏遮罩。
4. 结果：列表直接显示封面、作者、年份、来源、访问状态和一个最合理主行动。
5. 详情：在稳定内容区展示书目、简介、所有合法行动、复制与保存。
6. 私人书架：保存搜索结果，导入用户自己的 EPUB/PDF，本地重新打开或导出。
7. 错误与降级：单一数据源失败不白屏；无 Google Key 时继续使用 Open Library 与 Gutendex。

## 视觉系统

- 色彩：warm paper `#F2EEE5`、ink `#18221D`、forest `#163D31`、oxide `#A34731`、muted olive-gray；全部使用 CSS Token。
- 字体：系统无衬线负责界面和数字；Georgia/Charter 类衬线负责品牌与书名，不加载大字体依赖。
- 间距：8px 基础节奏；内容密度接近优质图书目录，避免空洞大留白。
- 圆角：输入和大面板 18–24px，小控件 8–12px；不用全页面统一胶囊。
- 边界：1px 低对比墨线和极轻纸张阴影；朱红只用于索引、警示和当前状态。
- 图标：统一 Lucide，文字与图标共同表达状态。
- 图像：只使用真实上游书封；无封面时使用排版式文本封面，不用假插画。
- 动效：搜索结果轻微上移淡入、按钮按压和详情切换；减少动态时全部关闭。

## 响应式

- 1440×900：首屏 58/42 工作台，搜索占主视觉；结果行可直接行动。
- 768×1024：首屏与工具单列，阅读路径图例压缩为横向标签；结果保持封面与主行动。
- 390×844：品牌和导航紧凑，搜索按钮全宽；结果主行动不被折叠，详情和书架改为单列。
- 主要触控目标至少 44×44px，正文不小于 16px，对比度不低于 4.5:1。
- 减少动态：禁用进入动画、平滑滚动和装饰位移。

## 状态设计

实现 default、hover、focus-visible、active、disabled、loading、empty、success、error、offline/fallback 和 local-storage-unavailable。

## 反复制约束

- 继承 P1 的质量门槛，不继承星空背景、严格灰阶、Poppins、Source Serif 4、52/48 固定布局、液态玻璃、电影字幕或 Three.js。
- 禁止通用紫蓝渐变、三列功能卡、全页居中、重复大圆角卡片和虚构社会证明。
- 卡片只代表一本书、一个本地文件或一个真实操作边界。
